# Feature Log - K 线时间与 24h 视图修正 - 时间：2025-12-08 14:18:11
- 负责人：Codex

## 背景 / 范围
- 前端 5d 视图把日线数据当作 24h 时间轴，显示 08:00；同时 K 线 x 轴时间格式需避免被解析为 UTC 偏移。

## 实施内容
1) 后端时间格式
   - K 线主/指数/备用路径的 `time` 改为 `"YYYY-MM-DD 00:00"`（本地日期字符串），避免 JS 解析为 UTC 并显示 08:00。
2) 前端 24h 视图判定
   - `StockChart`: 检测数据中是否含 “:” 视为盘中数据，只有盘中数据时才做 24h 过滤与时间轴格式化；日线数据不再误判为 24h，轴与 tooltip 使用日期。

## 测试
- `python -m pytest backend/tests/test_kline.py -q`（通过，保留既有 PytestReturnNotNoneWarning）。

## 影响 / 下一步
- 日线数据不再显示 08:00 偏移，5d 视图不会把日线压成单点时间轴；若后续接入分时数据，自动切换到 24h 视图仍生效。
